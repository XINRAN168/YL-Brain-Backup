## 流程详情

### 1. 项目启动流程

**适用场景**：任何新项目/任务的启动

**前置条件**：
- 任务目标已明确
- 资源已初步评估
- 相关方已识别

**六步启动流程**：
```
┌────────────────────────────────────────────────────┐
│                                                    │
│  Step 1: 明确项目背景                              │
│  ────────────────────                             │
│  □ 为什么做这个项目？                             │
│  □ 不做的后果是什么？                             │
│  □ 与公司战略的关联？                             │
│  输出：项目背景文档                                │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 2: 定义项目目标                            │
│  ──────────────────                               │
│  □ 最终交付物是什么？                             │
│  □ 成功标准是什么(SMART)？                       │
│  □ 质量标准是什么？                               │
│  输出：项目目标说明书                              │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 3: 识别相关方                               │
│  ──────────────────                              │
│  □ 发起人是谁？                                   │
│  □ 关键利益相关方？                               │
│  □ 团队成员？                                     │
│  □ 需要通知的人？                                 │
│  输出：相关方登记表                                │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 4: 制定里程碑计划                           │
│  ────────────────────                             │
│  □ 关键节点有哪些？                               │
│  □ 时间线是什么？                                 │
│  □ 重大风险事件点？                               │
│  输出：里程碑计划表                                │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 5: 资源与预算                              │
│  ────────────────                                │
│  □ 需要什么人？                                   │
│  □ 需要多少钱？                                   │
│  □ 需要什么工具/系统？                            │
│  输出：资源需求清单                                │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 6: 启动会议                                │
│  ────────────────                                │
│  □ 通报项目背景与目标                             │
│  □ 确认团队分工                                  │
│  □ 沟通工作方式                                  │
│  □ 约定沟通机制                                  │
│  输出：会议纪要、团队分工表                         │
│                                                    │
└────────────────────────────────────────────────────┘
```

**项目启动检查清单**：
```
□ 项目背景清晰 □ 目标SMART □ 相关方识别完整
□ 里程碑明确 □ 资源到位 □ 沟通机制约定 □ 启动会议完成
```

**输出物清单**：
- 项目背景文档
- 项目目标说明书
- 相关方登记表
- 里程碑计划表
- 资源需求清单
- 启动会议纪要

### 2. 项目执行管控流程

**适用场景**：项目实施过程中的日常管控

**前置条件**：项目已正式启动

**日常管控流程**：
```
每日站会(15分钟)：
┌────────────────────────────────────┐
│  昨天完成了什么？                   │
│  今天计划做什么？                   │
│  有什么阻碍？                       │
└────────────────────────────────────┘
        │
        ▼
周报汇总：
┌────────────────────────────────────┐
│  本周进度(绿/黄/红)                 │
│  风险与问题                         │
│  下周计划                           │
│  需要的支持                         │
└────────────────────────────────────┘
        │
        ▼
周例会检视：
┌────────────────────────────────────┐
│  进度偏差分析                       │
│  风险评估                           │
│  决策事项                           │
│  资源协调                           │
└────────────────────────────────────┘
        │
        ▼
里程碑验收：
┌────────────────────────────────────┐
│  交付物检查                         │
│  质量验收                           │
│  干系人确认                         │
└────────────────────────────────────┘
```

**进度偏差处理**：
```
发现偏差(>5%偏差)：
    │
    ▼
评估原因：
┌────────────────┬────────────────┐
│ 内部原因        │  外部原因       │
│ - 估计不准      │  - 需求变更     │
│ - 资源不足      │  - 依赖方延迟   │
│ - 技术难题      │  - 市场变化     │
└────────────────┴────────────────┘
        │
        ▼
制定纠偏措施：
□ 赶工(增加资源) □ 快速跟进(并行作业)
□ 范围调整 □ 质量妥协 □ 其他

重新评估：
□ 新计划可行？ □ 风险可接受？ □ 干系人同意？
        │
        ▼
  更新计划并执行
```

**质量控制要点**：
```
□ 每个阶段有交付物标准
□ 交付前有检查清单
□ 重要节点有评审
□ 问题有记录追踪
```

**风险管理流程**：
```
风险识别 ──► 风险评估 ──► 风险应对 ──► 风险监控
    │            │            │            │
    ▼            ▼            ▼            ▼
 头脑风暴   概率×影响   规避/转移    定期检视
 问题清单   优先级      /减轻/接受   触发应对
```

### 3. 项目复盘流程

**适用场景**：项目阶段性复盘或项目结束复盘

**复盘时机**：
- 项目关键里程碑后
- 项目完全结束后
- 项目终止时

**四步复盘法**：
```
┌────────────────────────────────────────────────────┐
│                                                    │
│  Step 1: 回顾目标                                  │
│  ───────────────                                  │
│  □ 最初的目标是什么？                              │
│  □ 现在的结果是什么？                             │
│  □ 目标是否达成？                                 │
│  □ 与目标的差距？                                  │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 2: 分析原因                                  │
│  ───────────────                                  │
│  □ 成功因素有哪些？(Keep)                         │
│  □ 失败因素有哪些？(Problem)                      │
│  □ 根本原因是什么？                               │
│  □ 5Why深入分析                                   │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 3: 总结规律                                  │
│  ────────────────                                 │
│  □ 学到了什么经验？                               │
│  □ 可以复用哪些做法？                             │
│  □ 需要停止哪些做法？                             │
│  □ 下次需要改进什么？                             │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 4: 行动计划                                  │
│  ────────────────                                 │
│  □ 下次同类项目如何改进？                         │
│  □ 具体改进行动是什么？                           │
│  □ 谁负责执行？                                   │
│  □ 何时完成？                                     │
│                                                    │
└────────────────────────────────────────────────────┘
```

**复盘会议准备**：
```
会前(提前1天)：
□ 收集项目数据 □ 整理文档 □ 通知参与者
□ 发送复盘问卷 □ 准备复盘框架

会中(1-2小时)：
□ 回顾目标 □ 坦诚分享 □ 深入分析 □ 形成共识

会后(3天内)：
□ 发送复盘报告 □ 跟踪行动计划 □ 更新知识库
```

**KISS复盘模板**：
```
K - Keep(继续保持)
   做得好的，要保持的好做法
   
I - Improve(需要改进)
   做得不好的，需要改进的地方
   
S - Stop(需要停止)
   不应该再做的事

S - Start(需要开始)
   以前没做，现在应该开始做的事
```

**检查清单**：
```
□ 所有关键干系人参与 □ 坦诚不推卸责任 □ 记录完整
□ 经验教训文档化 □ 行动计划有负责人 □ 纳入知识库
```
