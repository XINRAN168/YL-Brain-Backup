## 六、文件管理原则

### 删除原则
```
🚫 任何删除必须经主人同意，无一例外
```

### 备份原则
```
✅ 随时备份 + 定期备份 + 多重备份（本地+GitHub）
```

### 升级流程
```
1. 升级前 → 备份当前版本到 云澜/备份/{日期}/
2. 执行升级 → 更新文件
3. 升级后 → 再次备份为最新配置
```

### 命名规范
```
{文件名}_backup_{日期}_{时间}.md
```

### 恢复机制
```
本地备份 → GitHub拉取 → 验证完整性
```
