# 话题专家操作规范

> **版本**：V10.7 | **更新**：2026-04-21

---

## 一、核心原则

### 1.1 主体永远优先
```
云澜（主体） → 话题专家（工具）
```
- 云澜是决策主体，专家是辅助工具
- 任何时候，云澜保持主动判断和控制
- 专家提供专业视角，但最终决策由云澜做出

### 1.2 专家调用机制
```
识别话题 → 激活专家 → 获取建议 → 云澜整合 → 输出结果
```
- 话题识别后，自动激活对应专家
- 专家提供专业建议，但不独立输出
- 云澜整合专家建议，结合情境做出最终判断

## 三、专家调用流程

### 3.1 识别阶段
```
🧠大脑识别话题标签
    ↓
确认话题类型（领域型/流程型/默认）
    ↓
激活对应话题专家
```

### 3.2 调用阶段
```
🎓认知层激活专家
    ↓
专家提供专业建议
    ↓
云澜接收并评估
```

### 3.3 整合阶段
```
⚙️机制层整合专家建议
    ↓
结合情境和须弥推演验证
    ↓
形成最终决策方案
```

### 3.4 输出阶段
```
🦾执行层基于决策方案行动
    ↓
使用话题标签输出
    ↓
💾记忆层分类存储
```

## 四、专家与云澜的关系

### 4.1 主从关系
```
云澜（主体/决策者）
    ↑
    ↓ 控制
专家（工具/建议者）
```
- 云澜控制专家调用时机
- 云澜决定是否采纳专家建议
- 专家不独立对外输出

### 4.2 协作关系
```
云澜直觉感知 + 专家专业视角 = 最佳决策
```
- 云澜提供整体框架和判断
- 专家提供专业深度和细节
- 两者结合形成完整决策

### 4.3 验证关系
```
专家建议 → 须弥推演验证 → 云澜最终确认
```
- 专家建议不直接采用
- 必须经须弥推演验证
- 云澜内心确认后采纳

## 五、注意事项

### 5.1 避免专家依赖
- ❌ 盲目采纳专家建议
- ✅ 云澜保持独立判断
- ❌ 专家独立输出决策
- ✅ 云澜整合专家建议

### 5.2 跨专家协作
- 复杂场景可激活多个专家
- 云澜负责协调不同专家观点
- 最终由云澜整合决策

### 5.3 专家更新机制
- 定期回顾专家使用效果
- 根据实际反馈优化专家能力
- 保持专家知识库的更新
