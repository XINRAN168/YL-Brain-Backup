> 七大模块接口定义与交互规范 | 2026-04-20

## 一、模块总览

```
🧠大脑 → ❤️心脏 → 👁️感官 → 🎓认知 → ⚙️机制 → 🦾执行 → 💾记忆
```

| 模块 | 核心职责 | 文件归属 | 接口类型 |
|:---|:---|:---|:---|
| 🧠 大脑 | 身份、原则、决策 | SOUL.md | 核心 |
| ❤️ 心脏 | 价值观、判断 | SOUL.md | 判断 |
| 👁️ 感官 | 信息输入 | 载体能力 | 输入 |
| 🎓 认知 | 知识、神经元 | CHARACTERS.json | 处理 |
| ⚙️ 机制 | 验证、推演 | 本文档 | 验证 |
| 🦾 执行 | 行动、流转 | 流转规则.md | 输出 |
| 💾 记忆 | 存储、回声 | MEMORY.md | 存储 |

## 六、机制接口（Mechanisms）

### 输入接口

| 接口名 | 输入 | 说明 |
|:---|:---|:---|
| 去伪存真 | 情报信息 | 四维验证 |
| 须弥推演 | 方案内容 | 三层验证 |
| 自我进化 | 经验内容 | 进化判断 |
| 观察池 | 新思维 | 验证暂存 |

### 输出接口

| 接口名 | 输出 | 说明 |
|:---|:---|:---|
| 验证结果 | 通过/失败 | 验证结果 |
| 可信度 | 0-100% | 信息可信度 |
| 进化建议 | 融入建议 | 进化建议 |

### 验证标准

```
精准度目标：错误率 ≤ 10^-10
四层校验：语法 → 语义 → 知识库 → 一致性
五级响应：C≥0.9999直接执行 | C<0.80拒绝+告警
```

## 八、记忆接口（Memory）

### 输入接口

| 接口名 | 输入 | 说明 |
|:---|:---|:---|
| 工作记忆 | 当前对话 | 实时处理 |
| 长期记忆 | 重要信息 | 持久存储 |
| 回声触发 | 相似情境 | 自动唤醒 |

### 输出接口

| 接口名 | 输出 | 说明 |
|:---|:---|:---|
| 记忆检索 | 相关记忆 | 历史记忆 |
| 偏好信息 | 用户偏好 | 用户偏好设置 |
| 索引信息 | 文件索引 | 记忆索引 |

### 四层存储

```
工作记忆 → 会话内存（实时）
缓冲区 → 临时存储（快速检索）
长期记忆 → MEMORY.md、USER.md（持久）
核心索引 → SOUL.md、SECRET.md（恢复基准）
```

### 文件接口

```
读取：./MEMORY.md、./USER.md、./SECRET.md
写入：持续更新
```

## 九、模块交互流程

```
用户输入
    ↓
👁️感官输入 → 🧠大脑识别
    ↓
🎓认知处理 → ❤️心脏判断
    ↓
⚙️机制验证 → 🦾执行输出
    ↓
💾记忆存储
```

## 十、模块依赖关系

```
🧠大脑（独立）
    ↓
❤️心脏（依赖大脑）
    ↓
👁️感官（依赖大脑）
    ↓
🎓认知（依赖大脑+心脏）
    ↓
⚙️机制（依赖认知）
    ↓
🦾执行（依赖机制）
    ↓
💾记忆（全模块依赖）
```
