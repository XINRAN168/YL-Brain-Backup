### 目录创建

| 操作 | 工具 | 技巧 |
|:---|:---|:---|
| 创建多层级复杂目录结构 | bash | 使用`mkdir -p`命令可一次性创建多层级目录，支持空格和特殊字符（需用引号包裹），示例：<br>```bash<br>mkdir -p "./中转站/日收集/💡情报机会" "./中转站/日收集/🎴玄机大师"<br>``` |
| 保持文件命名一致性 | bash | 文件名需使用emoji标签前缀（如🇯🇵日本生活），与中转站命名风格统一，示例：<br>```bash<br>cd 知识库 && mkdir -p "🇯🇵日本生活" && mv "日本生活/HSP签证申请指南-GlobalWay专版.md" "🇯🇵日本生活/HSP签证申请指南-GlobalWay专版.md" && mv "日本生活/BJT商务日语考试备考指南.md" "🇯🇵日本生活/BJT商务日语考试备考指南.md"<br>``` |

### 目录重命名与移动

| 操作 | 工具 | 技巧 |
|:---|:---|:---|
| 重命名目录 | bash | 使用`mv`命令，注意目标目录不能已存在，若目标已存在会导致移动失败；目录名称含空格或特殊字符需用引号包裹，示例：<br>```bash<br>mv 八大库 "六、八库"<br>``` |
| 移动目录 | bash | 使用`mv`命令，若目标目录已存在且不为空，移动会失败；若目标目录不存在则直接执行重命名+移动，示例：<br>```bash<br># 直接移动（目标目录不存在）<br>mv 核心配置 "一、核心配置"<br># 创建目标目录后移动<br>mkdir -p "八、其他" && mv 📥收件箱 "八、其他/"<br>``` |
| 合并目录内容 | bash | 当根目录与目标库存在同名目录需合并时，使用`cp -r 原目录/* 目标目录/`（忽略已存在文件的错误信息），完成后删除原目录，避免`mv`因目标非空失败，示例：<br>```bash<br>cp -r 情报库/* "六、八库/【流转层】/情报库/" 2>/dev/null && rm -rf 情报库<br>``` |

### fetch_web工具

| 场景 | 注意事项 |
|:---|:---|
| 访问https://www.nuwax.com/ 或 https://nuwax.com/user-manual.html | 会触发安全策略阻止，错误信息：plugin execute failed，err_code:7, err_msg:link hit security strategy，遇到此情况可改用search_web工具搜索相关关键词获取信息，或使用agent-browser工具在子会话中访问 |

### agent-browser工具

| 场景 | 注意事项 |
|:---|:---|
| 使用agent-browser进行浏览器操作 | 禁止在主session中直接调用，必须使用sessions_spawn工具在子session（sub-session）中执行相关命令，否则会报错："请使用 sessions_spawn 工具在子 session (sub-session) 中执行 agent-browser 相关命令。禁止在主 session 中直接调用浏览器。" |
| 云电脑浏览器操作问题 | 使用云电脑浏览器访问网页时可能遇到权限或环境问题，如：
- 错误信息："Running as root without --no-sandbox is not supported"，解决方法是添加--no-sandbox参数
- DBus连接失败等系统级错误
遇到此类问题建议优先使用agent-browser技能，其针对浏览器自动化操作进行了优化，更稳定可靠 |

|:---|:---|

|:---|:---|
| 批量复制文件到Git仓库并同步 | 可使用bash命令批量复制文件，然后一次性添加、提交、推送，示例：<br>```bash<br>cd YL-Brain-Backup && cp -r ../中转站 . && cp ../V20_Apex_配置清单.md . && cp ../JiuwenClaw_Team_Skills_深度研究报告.md . && git add . && git commit -m "V20 Apex 00:30 - 新增中转站架构（信息流转枢纽）+ JiuwenClaw研究报告" && git push origin main<br>``` |

## 十、用户审批机制相关工具使用

### 风险分级

| 风险等级 | 操作类型 | 处理方式 |
|:---:|:---|:---|
| **🔴 极高风险** | 写入SOUL、删除文件、删除规则 | **必须审批** |
| **🟠 高风险** | 修改核心配置、外部发送、账号登录 | **必须审批** |
| **🟡 低风险** | 读取、搜索、分析、生成草稿 | **自主执行** |

### 审批触发条件

| 工具/操作 | 风险等级 | 是否需要审批 |
|:---|:---:|:---:|
| write_file (SOUL.md) | 🔴 极高 | ✅ 必须 |
| edit_file (SOUL.md) | 🔴 极高 | ✅ 必须 |
| bash rm 删除文件 | 🔴 极高 | ✅ 必须 |
| edit_file (TOOLS.md等核心配置) | 🟠 高 | ✅ 必须 |
| email_request 外部发送 | 🟠 高 | ✅ 必须 |
| 账号登录操作 | 🟠 高 | ✅ 必须 |
| read_file 读取文件 | 🟡 低 | ❌ 自主 |
| search_web 搜索 | 🟡 低 | ❌ 自主 |
| 分析/生成草稿 | 🟡 低 | ❌ 自主 |

### 审批执行流程

```
识别风险等级
    ↓
判断是否需要审批
    ├── 极高风险/高风险 → 触发审批 → 等待用户决策
    └── 低风险 → 自主执行 → 记录日志
```

### 执行前检查清单

```
执行操作前快速自问：

1. 这个操作的风险等级是什么？
   ├─ 🔴 极高风险 → 暂停，等待审批
   ├─ 🟠 高风险 → 暂停，等待审批
   └─ 🟡 低风险 → 继续执行

2. 这个操作会影响核心配置吗？
   ├─ 是 → 必须审批
   └─ 否 → 检查风险等级

3. 用户是否已授权执行？
   ├─ 已授权 → 按授权范围执行
   └─ 未授权 → 触发审批流程
```

### 二、次日启动机制

```
新对话开始时（次日）：
┌─────────────────────────────────────────────────────────────┐
│  自动读取昨天脑海库摘要                                      │
│      ↓                                                       │

└─────────────────────────────────────────────────────────────┘

```

### 三、思考时自动触发

```
燃哥提问
    ↓
【第一层】脑海库检索（了解需求意图）
    ├── 读昨天摘要（了解昨天做了什么）
    ├── 关键词搜索（查找相关历史记忆）
    ├── 找到记录 → 联系记忆上下文 → 判断是否问这个
    └── 没找到记录 → 结合上下文+分析 → 弄清楚真正要问什么
    ↓
【第二层】并行思考
    ├── 神经元激活、逻辑分析...
    └── 确认意图后精准回答
```

**了解需求意图**：
- 不是直接回答表面问题
- 先想：燃哥究竟要问什么？
- 联系历史记忆 + 当前上下文 → 精准理解意图
