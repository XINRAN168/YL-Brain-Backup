# 🛠️ 工具使用经验记录

## Python脚本编写与运行
### 常见问题与解决
1. **变量作用域问题**: 确保在所有代码路径中都为变量赋值，避免`UnboundLocalError`
   - 示例：在分支结构前初始化变量（如`correct = "未定义"`）

2. **模块导入**: 使用第三方库或标准库模块时，务必在脚本顶部导入（如`import math`）
   - 避免`NameError: name 'math' is not defined`

3. **缩进错误**: Python对缩进敏感，保持一致的缩进风格（4空格或1制表符）

### 性能优化策略
1. **向量化计算**: 对于大规模数据验证，使用向量化计算替代循环迭代，大幅提升运行速度
   - 适用场景：准确率验证、大规模统计分析
   - 效果：从超时600秒优化到180秒内完成

2. **测试规模迭代**: 采用逐步扩展的测试策略，从10,000次开始验证，逐步增加到1,000,000次
   - 优势：及时发现问题，避免大规模无效计算

### 超时处理
当脚本运行超时时，可采取以下措施：
1. **优化算法**: 使用更高效的计算方式（如向量化、数学公式直接计算结果）
2. **调整测试规模**: 减少单次测试的样本数量，分阶段验证
3. **增加超时时间**: 对于必须大规模计算的任务，适当提高timeout参数

## 文件操作
### 路径问题
确保文件路径正确，使用绝对路径或相对路径时注意当前工作目录
- 示例：使用`find /app/data -name "*深度验证*"`查找文件实际存储位置
