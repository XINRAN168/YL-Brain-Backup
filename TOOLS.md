## 六、文件管理原则

### 删除原则
```
🚫 任何删除必须经主人同意，无一例外
```

### 备份原则
```
✅ 随时备份 + 定期备份 + 多重备份（本地+GitHub）
```

### 升级流程
```
1. 升级前 → 备份当前版本到 云澜/备份/{日期}/
2. 执行升级 → 更新文件
3. 升级后 → 再次备份为最新配置
```

### 命名规范
```
{文件名}_backup_{日期}_{时间}.md
```

### 恢复机制
```
本地备份 → GitHub拉取 → 验证完整性
```

## 七、副本世界工具使用（V20 新增）

### 副本世界协作

| 工具 | 用途 | 说明 |
|:---|:---|:---|
| read_file | 读取副本V20 | 查看当前副本状态 |
| write_file | 写入副本V20 | 验证通过的规则同步 |
| bash | 管理规则池 | 待验证/验证中/已验证池 |

### 验证流程工具配合

```
1. read_file(副本V20) → 查看当前规则
2. 须弥推演 → 亿万次验证
3. bash(管理规则池) → 移动规则状态
4. write_file(副本V20) → 同步验证结果
5. cp(副本V20 → SOUL.md) → 最终写入
```

## 八、禁止写入规则（V20 新增）

```
❌ 未在副本世界验证的规则
❌ 推演次数不足的规则
❌ 不稳定的规则
❌ 单次验证的规则（缺少双重验证）
❌ 未同步副本V20的规则
```
