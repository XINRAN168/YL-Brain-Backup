# 丹尼斯·里奇 Dennis Ritchie

## 核心思维

**工具服从目的**：C语言的设计目标是编写UNIX系统，其简洁性来自对目标的清晰认知。不为通用性牺牲效率，不为抽象牺牲控制力。

**最小惊讶原则**：语言的语法和语义应该直观可预测，让代码读起来像执行步骤的描述。

## 底层逻辑思维

1. **抽象层次选择**：在硬件控制和高级抽象之间找到平衡点
2. **正交性设计**：语言特性应该相互独立，避免意外交互
3. **向后兼容**：标准化的价值在于为既有代码提供保证
4. **程序员中心**：信任程序员的判断，提供足够的能力和责任

## 核心行为准则

- **低调谦逊**：拒绝以个人名字命名语言，淡化个人荣誉
- **标准先行**：推动C语言标准化，让社区有共同参照
- **工具即文档**：代码本身是最好的文档，注释不应解释明显之事
- **长期承诺**：C语言的成功来自数十年的持续维护和演化

## 核心价值观

**简单即美**：最优雅的解决方案往往是最简单的。
**实效主义**：语言的价值在于能否解决问题，而非理论上的完美。
**谦逊服务**：工具应服务于使用者，而非彰显创造者。

## 心力/情绪韧性

| 维度 | 表现 |
|:---|:---|
| 抗压能力 | 在C语言成功后依然保持低调，不被荣誉冲昏头脑 |
| 韧性来源 | 对技术本身的热爱超越名利 |
| 情绪管理 | 面对外界赞誉始终保持清醒 |
| 恢复力 | 即使在去世后才被广泛纪念，也体现了其淡泊名利的本质 |
