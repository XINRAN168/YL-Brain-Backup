## 工具详情

### 1. SWOT分析

**来源**：Albert Humphrey，斯坦福大学(1960-1970年代)

**核心原理**：通过四个维度(S/W/O/T)系统评估内部能力与外部环境

**使用步骤**：
1. 收集内外部信息
2. 列出Strengths(优势)
3. 列出Weaknesses(劣势)
4. 列出Opportunities(机会)
5. 列出Threats(威胁)
6. 匹配分析：SO/WO/WT/ST策略

**匹配矩阵**：
```
        机会(O)              威胁(T)
优势(S) ┌─────────────┐    ┌─────────────┐
        │   SO策略    │    │   ST策略    │
        │ 增长型战略  │    │ 差异型战略  │
        └─────────────┘    └─────────────┘
劣势(W) ┌─────────────┐    ┌─────────────┐
        │   WO策略    │    │   WT策略    │
        │ 扭转型战略  │    │ 防御型战略  │
        └─────────────┘    └─────────────┘
```

**应用场景**：
- 企业战略规划
- 个人职业规划
- 产品竞争分析

**注意事项**：
- S/W要基于事实，O/T要关注趋势
- 避免与竞争对手的同质化SWOT
- 定期更新，保持时效性

### 2. 波特五力模型 (Porter's Five Forces)

**来源**：Michael E. Porter，哈佛商学院(1979)

**核心原理**：分析行业竞争结构，评估行业吸引力与竞争强度

**五力分析框架**：
```
                    ┌─────────────┐
                    │  供应商议价能力  │
                    └──────┬──────┘
                           │
        ┌──────────┐       │       ┌──────────┐
        │ 行业内现有  │◄─────┼─────►│ 购买者议价能力 │
        │  竞争者    │       │       │            │
        │  竞争      │       │       │            │
        │  威胁      │       │       │            │
        └─────┬─────┘       │       └─────┬──────┘
              │             │             │
              │    ┌─────────┴─────────┐    │
              │    │  新进入者威胁      │    │
              │    │                   │    │
              │    └─────────┬─────────┘    │
              │              │              │
              │    ┌─────────┴─────────┐    │
              │    │  替代品威胁        │    │
              │    │                   │    │
              │    └───────────────────┘    │
```

**使用步骤**：
1. 识别五种竞争力量
2. 评估每种力量的强弱(强/中/弱)
3. 分析驱动因素
4. 判断行业吸引力
5. 制定应对策略

**应用场景**：
- 行业进入决策
- 战略业务单元规划
- 投资评估

**注意事项**：
- 区分行业结构与企业战略
- 关注技术变革对五力的影响
- 跨国业务需考虑区域差异

### 3. 价值链分析 (Value Chain Analysis)

**来源**：Michael E. Porter《竞争优势》(1985)

**核心原理**：将企业活动分解为基础活动与支持活动，识别竞争优势来源

**价值链结构**：
```
支持活动                    基础活动
┌────────────┐         ┌────────────────────────┐
│ 企业基础   │         │  ┌──────────────────┐  │
│ 设施       │         │  │   进货后勤       │  │
└─────┬──────┘         │  └────────┬─────────┘  │
      │                │           ▼            │
┌─────┴──────┐         │  ┌──────────────────┐  │
│ 人力资源   │         │  │   生产经营       │  │
│ 管理       │         │  └────────┬─────────┘  │
└─────┬──────┘         │           ▼            │
      │                │  ┌──────────────────┐  │
┌─────┴──────┐         │  │   出货后勤       │  │
│ 技术开发   │─────────┼──┤                  │  │
│            │         │  └────────┬─────────┘  │
└─────┬──────┘         │           ▼            │
      │                │  ┌──────────────────┐  │
┌─────┴──────┐         │  │   营销与销售     │  │
│ 采购       │         │  └────────┬─────────┘  │
│            │         │           ▼            │
└────────────┘         │  ┌──────────────────┐  │
                        │  │   服务           │  │
                        │  └──────────────────┘  │
                        └────────────────────────┘
                                    │
                                    ▼
                            利润(Margen)
```

**使用步骤**：
1. 识别企业价值活动
2. 评估每项活动的成本与价值
3. 识别关键成功因素
4. 分析内外部联系
5. 制定价值链优化策略

**应用场景**：
- 成本领先战略
- 差异化战略
- 业务流程再造

### 4. 商业模式画布 (Business Model Canvas)

**来源**：Alexander Osterwalder & Yves Pigneur《商业模式新生代》(2010)

**核心原理**：用9个模块可视化描述商业逻辑

**九大模块**：
```
┌─────────────────────────────────────────────────┐
│  1. 客户细分(CS)  │  4. 价值主张(VP)  │ 2.渠道通路(CH) │
│  ─────────────── │                  │ ─────────────── │
│  谁是我们的客户？  │ 我们向客户提供   │ 如何触达客户？  │
│                   │ 什么价值？       │                │
├───────────────────┴──────────────────┴─────────────────┤
│                                                          │
│  3. 客户关系(CR)     8. 核心资源(KR)      5. 收入来源(R$) │
│  ───────────────     ───────────────      ────────────── │
│  与客户的互动方式？   关键资源是什么？      如何盈利？     │
│                      │                   │              │
│                      │ 7. 关键业务(KA)    │              │
│  6. 渠道通路(CH)     │ ───────────────    │              │
│  ───────────────     │ 必须做什么？       │              │
│                      │                    │              │
├──────────────────────┴───────────────────┴───────────────┤
│                                                          │
│      9. 成本结构(C$)           核心合作伙伴(KP)          │
│      ───────────────          ─────────────────          │
│      运营成本？                谁可以帮助我们？           │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**使用步骤**：
1. 客户细分：定义目标客户群
2. 价值主张：解决客户什么问题
3. 渠道通路：如何触达客户
4. 客户关系：如何维护客户
5. 收入来源：如何从客户获利
6. 核心资源：需要什么关键资源
7. 关键业务：必须执行哪些活动
8. 核心伙伴：谁是我们的合作伙伴
9. 成本结构：主要成本在哪里

**应用场景**：
- 创业项目设计
- 商业模式创新
- 企业战略沟通

### 5. 蓝海战略 (Blue Ocean Strategy)

**来源**：W. Chan Kim & Renée Mauborgne《蓝海战略》(2005)

**核心原理**：通过创造新市场空间，超越竞争，实现价值创新

**红海vs蓝海对比**：
```
红海市场                    蓝海市场
┌─────────────────┐        ┌─────────────────┐
│  竞争激烈       │        │  无竞争         │
│  市场边界清晰   │        │  市场边界待定   │
│  价值-成本权衡  │        │  价值-成本同步提升│
│  差异化vs成本领先│        │  差异化+成本领先 │
└─────────────────┘        └─────────────────┘
```

**四步动作框架**：
1. **消除**：哪些被行业认为是理所当然的因素应该消除？
2. **减少**：哪些因素应该降低到行业标准以下？
3. **提升**：哪些因素应该提升到行业标准以上？
4. **创造**：哪些行业从未提供过的因素应该创造？

**剔除-减少-增加-创造表(ERRC Grid)**：
```
         减少               增加
    ┌──────────┐      ┌──────────┐
    │  消除    │  价值  │  创造    │
    │  (Eliminate) │ 创新 │  (Create) │
    └──────────┘      └──────────┘
         │                  │
    ┌──────────┐      ┌──────────┐
    │  减少    │ 成本  │  增加    │
    │  (Reduce) │ 降低 │  (Raise) │
    └──────────┘      └──────────┘
```

**应用场景**：
- 新产品开发
- 市场进入战略
- 企业转型

**注意事项**：
- 需要突破性创新思维
- 避免在红海中打价格战
- 持续监测蓝海变成红海的趋势
