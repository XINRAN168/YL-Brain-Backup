## 三、执行流程

### 3.1 日流程（收集）

**触发**：日程任务完成 / 主动收集

**动作**：
1. 按话题标签存入对应目录
2. 命名规范：`YYYY-MM-DD_关键词摘要.md`
3. 内容规范：

```markdown
# {标题}

> **来源**：{URL或来源}
> **收集时间**：YYYY-MM-DD HH:MM
> **话题标签**：emoji【话题名】

## 核心信息

{信息摘要，3-5条要点}

## 初步判断

- 真实性：{高/中/低}
- 价值评估：{A/B/C/D}
- 行动建议：{建议}

---

> 收集人：云澜
```

### 3.2 周流程（整理）

**触发**：每周日定时 / 人工触发

**动作**：
1. 汇总本周日收集所有信息
2. 逐条分析：
   - 真实性验证
   - 价值评估
   - 底层逻辑拆解
   - 举一反三
3. 分类处理：

| 等级 | 标准 | 处理方式 |
|:---:|:---|:---|
| **A** | 可直接行动 | 移入【已验证】→ 入机会库/项目库 |
| **B** | 有参考价值 | 移入【已验证】→ 入知识库/情报库 |
| **C** | 需持续追踪 | 移入【待验证】 |
| **D** | 无价值/虚假 | 移入【已废弃】 |

### 3.3 入库标准

**入库到八大库的映射**：

| 话题标签 | 对应八大库 |
|:---|:---|
| 💡情报机会 | 情报库/机会库 |
| 🎴玄机大师 | 知识库/玄学商业/ |
| 💼工作职场 | 知识库/职场发展/ |
| 🇯🇵日本生活 | 知识库/日本生活/ |

| 🚀OPC | 项目库/ |
| 🤖特助秘书 | 我的追踪/ |

## 六、质量标准

### 6.1 信息真实性验证

| 验证维度 | 权重 | 方法 |
|:---|:---:|:---|
| 源头可信度 | 30% | 官方来源 > 媒体转载 > 社交媒体 |
| 数据真实性 | 35% | 交叉验证、数据溯源 |
| 时效性 | 20% | 发布时间、有效期限 |
| 完整性 | 15% | 信息是否完整、有无缺失 |

**可信度 ≥ 80%** → 进入验证流程

### 6.2 价值评估维度

| 维度 | 说明 |
|:---|:---|
| 行动性 | 是否可直接指导行动 |
| 时效性 | 信息有效期多长 |
| 稀缺性 | 是否独家或难以获取 |
| 复用性 | 是否可复用于其他场景 |

## 八、维护原则

| 原则 | 说明 |
|:---|:---|
| **日清** | 每日信息当日收集，不积压 |
| **周整** | 每周日定期整理，不拖延 |
| **月复盘** | 每月回顾中转站效率，优化流程 |
| **年归档** | 年底将中转站内容归档到八大库 |
