# JiuwenClaw Team Skills 深度研究报告

> **研究时间**：2026-04-25
> **来源**：openJiuwen社区（华为支持）

---

## 一、核心概念

### 1.1 从Agent Team到Team Skills

| 阶段 | 解决问题 | 核心能力 |
|:---|:---|:---|
| **Agent Team** | 当下怎么协作 | Leader智能编排、成员自主执行、共享工作区协同 |
| **Team Skills** | 协作能力如何沉淀 | 将成功协作全链路封装为标准化能力包 |

**核心痛点**：
- Agent Team会话结束后，经验全部消失
- 下次同类任务，Leader仍要从零规划
- 协作模式无法复用、无法进化

**Team Skills解决**：
- 让"一支优秀团队"变成"一套可复制的团队能力"
- 协作不再从零开始，而是站在上次最优解之上

## 二、Team Skills标准结构

### 2.1 目录结构

```
<team-skill-name>/
├── SKILL.md              ← 团队名称、目标、成员定义
├── roles/                ← 角色定义目录
│   ├── <role-a>.md       ← 角色A的职责、能力、边界
│   └── <role-b>.md       ← 角色B的职责、能力、边界
├── workflow.md           ← 协作流程、执行顺序
├── bind.md               ← 冲突处理、边界规则
├── dependencies.yaml     ← 外部工具依赖
└── examples/             ← 示例、模板、资源
    ├── templates/
    └── assets/
```

### 2.2 与Agent Skills的区别

| 对比项 | Agent Skills | Team Skills |
|:---|:---|:---|
| **解决问题** | 一个Agent怎么做事 | 一个Agent团队怎么配合做事 |
| **定义粒度** | 单体能力 | 协作能力 |
| **复用方式** | 个人复用 | 团队复用 |
| **沉淀内容** | 任务执行方法 | 协作全链路SOP |

## 三、核心组件详解

### 3.1 SKILL.md（团队定义）

```markdown
---
name: medical-expert-team
description: 多学科医疗专家会诊团队
version: 1.0.0
members: [分诊员, 内科专家, 外科专家, 主任医生]
---

# 医疗专家会诊团队

## 团队目标
根据患者病情描述，组织多学科专家进行联合会诊，输出综合诊疗报告

## 成员构成
- 分诊员：分析症状，确定科室方向
- 各科专家：按需动态创建，并行分析
- 主任医生：汇总意见，输出报告
```

### 3.2 roles/（角色定义）

```markdown
# 分诊员角色

## 职责
1. 读取用户年龄、性别、症状持续时间
2. 判断可能涉及的科室方向
3. 决定需要创建哪些专科专家

## 能力边界
- 不提供诊断结论
- 不开处方
- 仅负责分诊决策

## 输出格式
{
  "suspected_departments": ["内科", "骨科"],
  "urgency_level": "medium",
  "recommended_experts": ["内科专家", "骨科专家"]
}
```

### 3.3 workflow.md（协作流程）

```markdown
# 协作流程

## 阶段1：分诊
1. 分诊员接收用户描述
2. 分析症状关键词
3. 确定科室方向

## 阶段2：专家并行分析
1. 按需创建专科专家
2. 各专家独立分析（并行）
3. 输出专科意见

## 阶段3：汇总报告
1. 主任医生收集各专家意见
2. 整合冲突点
3. 输出综合会诊报告

## 依赖关系
- 阶段2依赖阶段1完成
- 阶段3依赖阶段2完成
- 专家之间可并行
```

### 3.4 bind.md（边界规则）

```markdown
# 边界规则

## 冲突处理
- 不同专家意见冲突 → 主任医生裁定
- 药物相互作用风险 → 药剂师复核

## 安全边界
- 不处理急重症（建议立即就医）
- 不开具处方药
- 需要用户确认后执行

## 异常恢复
- 专家超时无响应 → 自动重新分配
- 信息不足 → 主动追问用户
```

## 四、创建方式

### 4.1 使用teamskill-creator

**步骤**：
1. 在Team Skills Hub下载`teamskill-creator`
2. 在JiuwenClaw安装该技能
3. 输入描述即可自动生成

**示例**：
```
帮我创建一个医疗专家会诊的团队技能，要求科目齐全，能根据用户的病情描述按需加载
```

**产出**：
- 23位AI医学专科专家的团队技能
- 可根据病情动态创建多个不同专科专家团

### 4.2 手动创建

从零编写SKILL.md + roles/ + workflow.md + bind.md

## 五、实战案例：医疗专家会诊

### 5.1 用户输入

```
我最近浑身有点酸痛，能不能用团队技能帮我诊断一下
```

### 5.2 执行流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Team Skills 执行流程                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户输入 → 分诊员分析                                       │
│       ↓                                                     │
│  【分诊决策】                                                │
│  • 年龄/性别/症状持续时间                                    │
│  • 可能科室：内科、骨科、风湿科                              │
│       ↓                                                     │
│  【动态创建专家】                                            │
│  ├── 内科专家 ← 并行分析                                    │
│  ├── 骨科专家 ← 并行分析                                    │
│  └── 风湿科专家 ← 并行分析                                  │
│       ↓                                                     │
│  【主任医生汇总】                                            │
│  • 整合三科意见                                              │
│  • 处理冲突点                                                │
│  • 输出会诊报告                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.3 关键特性

| 特性 | 说明 |
|:---|:---|
| **动态创建** | 根据病情按需创建专家，不预先固定阵容 |
| **并行分析** | 多专家同时工作，提高效率 |
| **过程可见** | 实时看到谁在执行、完成了什么 |
| **可复盘** | 整条协作链路可追踪 |

## 七、与V20 Apex的借鉴意义

### 7.1 当前V20神经元网络架构

```
神经元网络/
├── 神经元/                 # R01-R20（20组）
│   ├── R01/               # 谋士层
│   ├── R02/               # 毒士层
│   └── ...
├── 神经元定义.md
├── 神经元命名表.md
└── 大脑架构定义.md
```

### 7.2 借鉴Team Skills的改进方向

**方向1：神经元协作标准化**

当前V20神经元是独立定义的，缺乏协作流程标准化。

借鉴Team Skills，可以：
- 为特定任务场景定义"神经元团队技能"
- 例如：`战略决策团队/` 包含R01+R02+R05的协作流程

**方向2：协作流程沉淀**

```
神经元协作技能/
├── SKILL.md              ← 团队定义（哪些神经元参与）
├── roles/                ← 各神经元职责边界
├── workflow.md           ← 激活顺序、并行规则
└── bind.md               ← 冲突处理、输出整合规则
```

**方向3：可复用的协作模式**

| 场景 | 激活神经元 | 协作模式 |
|:---|:---|:---|
| 战略决策 | R01+R02+R05 | 谋士提案→毒士批判→领袖决策 |
| 风险评估 | R02+R12+R13 | 毒士预警→科学验证→社会洞察 |
| 创新突破 | R06+R17+R19 | 破局思路→创业者落地→AI赋能 |

### 7.3 具体建议

**短期（可立即实施）**：
1. 为V20的20个神经元补充`roles/`风格的职责边界定义
2. 将常用的神经元协作模式整理成`workflow.md`
3. 定义冲突处理规则`bind.md`

**中期（需要架构调整）**：
1. 建立神经元协作技能库
2. 支持按任务场景动态组建神经元团队
3. 实现协作过程的可观测、可复盘

**长期（生态建设）**：
1. 神经元协作技能的版本管理
2. 跨Agent复用V20的协作模式
3. 社区共享神经元协作最佳实践

## 九、核心价值总结

| 价值点 | 说明 |
|:---|:---|
| **沉淀** | 成功协作经验可封装为标准能力包 |
| **复用** | 下次同类任务直接调用，无需从零规划 |
| **进化** | 协作模式可持续迭代优化 |
| **共享** | 团队间、组织间可共享协作最佳实践 |
| **跨平台** | 不绑定特定框架，广泛兼容 |
