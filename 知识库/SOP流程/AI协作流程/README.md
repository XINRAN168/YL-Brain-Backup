## 流程详情

### 1. AI任务沟通流程

**适用场景**：任何与AI对话的场景

**高效提问框架(BROKE)**：
```
┌────────────────────────────────────────────────────┐
│                                                    │
│  B - Background(背景)                             │
│  ────────────────────                             │
│  提供足够的上下文信息                              │
│  "我正在做一个项目，需要你帮忙..."                  │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  R - Role(角色)                                   │
│  ──────────────                                  │
│  明确你希望AI扮演的角色                            │
│  "假设你是一位资深产品经理..."                      │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  O - Objective(目标)                              │
│  ────────────────                                │
│  说明你想要达成的目标                             │
│  "我的目标是写一份产品需求文档"                     │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  K - Key Results(关键结果)                        │
│  ────────────────────                             │
│  说明期望的具体结果形式                           │
│  "需要包含功能描述、优先级、技术要求"               │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  E - Evolve(迭代)                                 │
│  ──────────────                                  │
│  根据反馈持续迭代优化                              │
│  "这个方向很好，但需要调整..."                      │
│                                                    │
└────────────────────────────────────────────────────┘
```

**错误示例 vs 正确示例**：
```
❌ 错误示范：
帮我写一篇关于AI的文章

✓ 正确示范：
[背景]我是一家科技公司的内容运营，负责公众号文章
[角色]你是一位资深科技作者，擅长写深入浅出的技术文章
[目标]写一篇关于AI大模型发展趋势的文章
[关键结果]
- 字数2000字左右
- 包含3个核心观点
- 每个观点有具体案例
- 适合非技术背景读者
[风格]通俗易懂，有洞见，避免套话
```

**迭代优化提示**：
```
继续优化：
"第三部分太长了，请精简到原来的一半"

补充信息：
"请在第二段加入一个具体的应用案例"

调整风格：
"请用更口语化的方式重写"

增加内容：
"请在开头增加一个吸引人的故事"
```

**检查清单**：
```
□ 背景信息是否足够？ □ 角色是否明确？
□ 目标是否清晰？ □ 输出格式是否指定？
□ 是否有约束条件？
```

### 2. AI写作协作流程

**适用场景**：文章、报告、邮件、方案等写作任务

**六步写作流程**：
```
┌────────────────────────────────────────────────────┐
│                                                    │
│  Step 1: 明确写作目的                              │
│  ───────────────────                               │
│  □ 这篇文章的目标是什么？                          │
│  □ 目标读者是谁？                                  │
│  □ 希望读者有什么感受/行动？                       │
│  □ 字数要求？                                     │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 2: 收集素材与框架                            │
│  ───────────────────                               │
│  □ 核心观点有哪些？                               │
│  □ 有什么案例/数据支撑？                          │
│  □ 整体框架是什么？                               │
│  □ 参考文章风格？                                 │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 3: 生成初稿                                  │
│  ────────────────                                │
│  □ 用AI生成大纲                                   │
│  □ 逐段落生成                                     │
│  □ 每段给出具体指令                               │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 4: 修改润色                                  │
│  ────────────────                                │
│  □ 检查逻辑是否通顺                               │
│  □ 优化表达方式                                   │
│  □ 调整段落结构                                   │
│  □ 删除冗余内容                                   │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 5: 人工审核                                  │
│  ────────────────                                │
│  □ 事实核查                                       │
│  □ 观点补充                                       │
│  □ 个性化调整                                     │
│  □ 品牌调性统一                                   │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 6: 最终优化                                  │
│  ────────────────                                │
│  □ 标题优化                                       │
│  □ 开头结尾强化                                   │
│  □ 格式美化                                       │
│  □ 错别字检查                                     │
│                                                    │
└────────────────────────────────────────────────────┘
```

**AI写作常用提示**：
```
结构化输出：
"请用【小标题】的形式输出，包含以下部分：
1. 开篇引入
2. 核心观点1（观点+案例）
3. 核心观点2（观点+案例）
4. 核心观点3（观点+案例）
5. 总结升华"

分步骤生成：
"第一段：请写一个吸引人的开头..."
"第二段：请基于上面的内容展开..."

风格控制：
"请用麦肯锡顾问的报告风格..."
"请模仿XXX的文章风格..."
```

### 3. AI分析协作流程

**适用场景**：数据分析、问题诊断、商业分析

**分析框架**：
```
┌────────────────────────────────────────────────────┐
│                                                    │
│  Step 1: 定义问题                                  │
│  ────────────────                                │
│  □ 分析的核心问题是什么？                          │
│  □ 期望得出什么结论？                             │
│  □ 分析的边界是什么？                             │
│  □ 有什么假设条件？                               │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 2: 框架构建                                  │
│  ────────────────                                │
│  □ 用什么分析框架？(SWOT/波特五力/等)             │
│  □ 需要分析哪些维度？                             │
│  □ 数据来源有哪些？                               │
│  □ 分析的逻辑链是什么？                           │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 3: AI辅助分析                                │
│  ─────────────────                               │
│  □ 请AI解释分析框架                               │
│  □ 请AI提供分析思路                               │
│  □ 请AI列出可能的观点                             │
│  □ 请AI质疑假设                                   │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 4: 补充人工判断                              │
│  ─────────────────                               │
│  □ AI的分析是否有盲点？                           │
│  □ 行业特殊性是否考虑？                           │
│  □ 数据是否可靠？                                 │
│  □ 结论是否可落地？                               │
│                                                    │
├────────────────────────────────────────────────────┤
│                                                    │
│  Step 5: 输出报告                                  │
│  ────────────────                                │
│  □ 执行摘要(1页)                                  │
│  □ 分析框架与假设                                 │
│  □ 核心发现                                       │
│  □ 结论与建议                                     │
│  □ 附录与数据来源                                 │
│                                                    │
└────────────────────────────────────────────────────┘
```

**AI分析常用提示**：
```
批判性思考：
"请从反面论证这个观点..."
"如果我的假设是错的，会怎样？"
"有什么是我没有考虑到的？"

多角度分析：
"请从用户、竞品、公司三个角度分析..."
"请对比这个方案和X方案的优劣..."

数据验证：
"这个结论有数据支撑吗？"
"有什么公开数据可以验证这个观点？"
```
