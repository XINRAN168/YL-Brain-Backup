## 工具详情

### 1. PREP法则

**来源**：麦肯锡咨询公司培训方法

**核心原理**：先说结论，再说理由，让对方快速抓住重点

**结构图**：
```
    ┌─────────────────────────────────────────┐
    │                                         │
    │  P(Point)     观点/结论                 │
    │  ─────────────────────────────────────  │
    │  "我认为..."                             │
    │  "我的建议是..."                         │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  R(Reason)     理由/原因                 │
    │  ─────────────────────────────────────  │
    │  "理由有三点：第一..."                    │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  E(Example)    具体例子                 │
    │  ─────────────────────────────────────  │
    │  "比如..."                               │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  P(Point)      重申结论                 │
    │  ─────────────────────────────────────  │
    │  "所以我的结论是..."                      │
    │                                         │
    └─────────────────────────────────────────┘
```

**使用步骤**：
1. **Point(结论)**：先说出核心观点
2. **Reason(理由)**：列出2-3个支撑理由
3. **Example(例子)**：用具体案例说明
4. **Point(结论)**：再次强调核心观点

**应用场景**：
- 会议发言
- 工作汇报
- 即兴演讲
- 邮件撰写

**案例示范**：
```
P(观点)：我觉得我们应该推迟上线两周
R(理由)：原因有三：1)还有两个bug未修复 2)测试覆盖率不足 3)运营准备未完成
E(例子)：上次类似项目仓促上线，出现了数据丢失问题
P(结论)：所以建议延期，确保质量后再上线
```

**注意事项**：
- 结论要简洁明确
- 理由要与结论强相关
- 例子要具体生动

### 2. STAR法则

**来源**：跨国企业面试培训体系

**核心原理**：用四要素结构化描述经历，让故事有血有肉

**结构图**：
```
    ┌─────────────────────────────────────────┐
    │                                         │
    │  S(Situation)   情境                   │
    │  ─────────────────────────────────────  │
    │  当时的具体背景是什么？                   │
    │  时间、地点、角色、任务                   │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  T(Task)        任务                   │
    │  ─────────────────────────────────────  │
    │  你面临的任务/挑战/目标是什么？           │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  A(Action)      行动                   │
    │  ─────────────────────────────────────  │
    │  你具体采取了什么行动？                   │
    │  重点是你"我"做了什么                    │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  R(Result)      结果                   │
    │  ─────────────────────────────────────  │
    │  最终结果如何？                          │
    │  最好量化(data-driven)                 │
    │                                         │
    └─────────────────────────────────────────┘
```

**使用步骤**：
1. **Situation**：描述背景
2. **Task**：明确任务
3. **Action**：详述行动
4. **Result**：给出结果

**案例示范(面试)**：
```
面试问题：请分享一个你克服困难完成项目的经历

S(情境)：2023年Q2，我负责的电商APP上线项目遇到严重困难
         原计划3个月上线，但核心开发突然离职

T(任务)：我需要在2个月内完成项目上线，不能延期

A(行动)：1) 重新评估技术方案，砍掉非核心功能
         2) 与外包团队合作补充开发资源
         3) 每天站立会议同步进度
         4) 自己学习关键技术亲自测试

R(结果)：项目如期上线，首月DAU 10万+，获得App Store推荐
         这次经历让我学会了危机管理
```

**注意事项**：
- A(Action)是核心，要详细描述
- R(Result)要正面积极
- 可以适当夸大挑战增加戏剧性
- 准备多个STAR故事应对不同问题

### 3. 非暴力沟通 (Nonviolent Communication)

**来源**：Marshall Rosenberg(马歇尔·卢森堡)，临床心理学(1960年代)

**核心原理**：诚实地表达自己，共情地理解他人，建立和谐人际关系

**沟通四要素**：
```
    ┌─────────────────────────────────────────┐
    │                                         │
    │  观察(Observation)  不评判              │
    │  ─────────────────────────────────────  │
    │  "我看到/听到..."                        │
    │  "当...时"                              │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  感受(Feeling)     表达情感            │
    │  ─────────────────────────────────────  │
    │  "我感到..."                             │
    │  (而非"我觉得是")                         │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  需要(Need)         探索需求            │
    │  ─────────────────────────────────────  │
    │  "因为我需要/看重..."                     │
    │                                         │
    ├─────────────────────────────────────────┤
    │                                         │
    │  请求(Request)      具体行动            │
    │  ─────────────────────────────────────  │
    │  "你是否愿意..."                         │
    │  "我希望..."                             │
    │                                         │
    └─────────────────────────────────────────┘
```

**暴力沟通的四种形式(异化的沟通方式)**：
1. **道德评判**：用道德标准评判他人
2. **比较**：与他人比较
3. **回避责任**：用"不得不"推卸责任
4. **要求**：命令式表达

**使用示例**：
```
❌ 暴力沟通：
"你怎么总是这么自私！从来不顾及别人的感受！"

✓ 非暴力沟通：
"当我看到你没有回复我的消息(观察)，
 我感到有些担心和失落(感受)，
 因为我很看重我们之间的联系(需要)，
 你是否愿意下次方便的时候回复我一下？(请求)
"
```

**案例(职场冲突)**：
```
❌ 暴力沟通：
"你这样做完全不对，太不专业了！"

✓ 非暴力沟通：
"当我看到报告里有三个数据错误(观察)，
 我感到有些失望(感受)，
 因为我需要报告准确无误才能向上级汇报(需要)，
 你是否愿意花半小时再检查一遍？(请求)
"
```

**应用场景**：
- 家庭沟通
- 职场协作
- 冲突解决
- 客户服务

**注意事项**：
- 区分观察与评判
- 感受是情绪词汇
- 请求要具体而非命令
- 需要持续练习
