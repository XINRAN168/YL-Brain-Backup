## 工具详情

### 1. 5Why分析法 (5 Whys)

**来源**：丰田佐吉，丰田生产系统(1930年代)

**核心原理**：通过连续追问"为什么"，追溯到问题的根本原因

**使用步骤**：
1. 明确问题现象
2. 问"为什么这个问题发生？"
3. 记录回答
4. 对回答继续问"为什么？"
5. 重复直到找到根本原因(通常5次左右)

**示例：为什么机器停了？**
```
问题：机器停了
Why1: 为什么停了？ → 超负荷保险丝烧断
Why2: 为什么超负荷？ → 轴承润滑不足
Why3: 为什么润滑不足？ → 润滑油泵压不上油
Why4: 为什么泵压不上油？ → 泵轴磨损
Why5: 为什么磨损？ → 没有过滤器，金属屑进入
                        ↑
                   根本原因！
```

**解决方案**：安装过滤器

**应用场景**：
- 质量问题分析
- 故障诊断
- 流程改进

**注意事项**：
- 避免跳步或表面原因
- 每步回答要有事实依据
- 允许多条因果链

### 2. 鱼骨图 (Fishbone Diagram)

**来源**：Kaoru Ishikawa(石川馨)，日本质量管理(1960年代)

**核心原理**：将问题原因按类别归类，找出所有潜在因素

**结构图**：
```
                    ┌─────────────────────────┐
                    │                         │
                    ▼                         │
    ┌─────────┐  ┌─────────┐    ┌──────────┐  │
    │  人员   │  │   方法   │    │   机器   │  │
    │  Man    │  │  Method  │    │ Machine  │  │
    └────┬────┘  └────┬────┘    └────┬─────┘  │
         │            │              │        │
─────────┼────────────┼──────────────┼────────
         │            │              │        主骨
         │            │              │        │
    ┌────▼────┐  ┌────▼────┐   ┌────▼─────┐  │
    │小原因1  │  │小原因3  │   │小原因5   │  │
    │小原因2  │  │小原因4  │   │小原因6   │  │
    └─────────┘  └─────────┘   └──────────┘  │
                    │                         │
              ┌─────┼─────┐                   │
              │     │     │                   │
         ┌────▼┐┌──▼───┐┌─▼─────┐            │
         │材料  ││测量  ││环境   │            │
         │Material││Measurement││Environment│
         └─────┘└──────┘└───────┘            │
                                              │
──────────────────────────────────────────────
                         问题
                      (效果/结果)
```

**六大类别(6M)**：
1. Man(人员) - 人为因素
2. Machine(机器) - 设备工具
3. Material(材料) - 原材料
4. Method(方法) - 流程工艺
5. Measurement(测量) - 检测标准
6. Environment(环境) - 工作环境

**使用步骤**：
1. 明确问题(写在中骨右端)
2. 确定主要类别
3. 每个类别头脑风暴原因
4. 追问每个原因的可能子原因
5. 识别最可能的主要原因

**应用场景**：
- 质量问题分析
- 根因分析会议
- 原因归类整理

**注意事项**：
- 避免在骨头上直接写解决方案
- 原因要可操作
- 配合5Why使用效果更佳

### 3. MECE原则 (Mutually Exclusive, Collectively Exhaustive)

**来源**：Barbara Minto《金字塔原理》(1973)

**核心原理**：将复杂问题分解为相互独立、完全穷尽的子问题

**MECE定义**：
```
Mutually Exclusive(相互独立)     Collectively Exhaustive(完全穷尽)
┌──────────────────────┐        ┌──────────────────────┐
│   A    │    B    │    │        │  ┌─────┬─────┬─────┐ │
│  区域1 │   区域2  │    │        │  │  A  │  B  │  C  │ │
│        │         │    │        │  │     │     │     │ │
│  A∩B=∅ │ 不重叠  │    │        │  ├─────┼─────┼─────┤ │
└──────────────────────┘        │  │ 100% 覆盖  │ │
                                 │  └─────┴─────┴─────┘ │
```

**分解方法**：
1. **公式法**：Revenue = Price × Quantity
2. **流程法**：阶段、步骤、生命周期
3. **要素法**：人员、设备、物料、方法、环境
4. **分类法**：内/外、上/下、好/坏

**示例(市场细分)**：
```
MECE分解：
├── 地域：华北、华东、华南、华中、西部
├── 年龄：18-25、26-35、36-45、46+
├── 收入：低收入、中低收入、中收入、高收入
└── 行为：从不购买、偶尔购买、经常购买
```

**应用场景**：
- 问题拆解
- 报告结构
- 商业分析

**注意事项**：
- 不要过度分解(一般2-5层)
- 每层换一个维度
- 检验：各部分之和是否等于整体

### 4. 金字塔原理 (Pyramid Principle)

**来源**：Barbara Minto，麦肯锡顾问方法(1973)

**核心原理**：结论先行，以上统下，归纳分组，逻辑递进

**结构图**：
```
                    ┌─────────┐
                    │ 结论    │  ← 核心观点(最重要)
                    └────┬────┘
              ┌───────────┼───────────┐
              ▼           ▼           ▼
         ┌────────┐ ┌────────┐ ┌────────┐
         │ 理由1  │ │ 理由2  │ │ 理由3  │  ← 支持性论点
         └────┬───┘ └───┬────┘ └───┬────┘
        ┌─────┼─────┐   │          │
        ▼     ▼     ▼   ▼          ▼
     ┌───┐ ┌───┐ ┌───┐│数据│     │数据│   ← 具体事实/数据
     │事实│ │事实│ │事实│└───┘     └───┘
     └───┘ └───┘ └───┘│                │
                      └────────────────┘
```

**两种论证方式**：
```
演绎推理(逻辑递进)              归纳推理(归纳分组)
                               
大前提 → 小前提 → 结论          ┌─────┐
                               ┌─┤论据1├──┐
A是B → C是A → C是B            │ └─────┘   │
                               │ ┌─────┐   │
                               ├─┤论据2├──▶结论
                               │ └─────┘   │
                               │ ┌─────┐   │
                               └─┤论据3├──┘
                                 └─────┘
```

**使用步骤**：
1. 明确你的核心结论
2. 列出3-7个支持理由
3. 每个理由找出2-3个事实
4. 检查逻辑是否通顺
5. 组织成金字塔结构

**SCQA框架(情境-冲突-问题-答案)**：
```
S(Situation)情境：设定背景，大家都认可的事实
C(Complication)冲突：情况变化，出现问题
Q(Question)问题：核心疑问，怎么办？
A(Answer)答案：你的解决方案
```

**应用场景**：
- 商业报告
- 演讲结构
- 邮件沟通

**注意事项**：
- 结论先行，减少阅读时间
- 每组理由不超过7个
- 逻辑关系要清晰
